# Inbox 归档

## 执行步骤

### 1. 扫描收件箱

使用 Glob 工具扫描 `00-Inbox/**/*.md`，列出所有待处理的笔记文件。

如果收件箱为空，告知用户"收件箱已清空，没有需要处理的笔记"并结束。

### 2. 逐个处理

对每个文件：

1. **读取文件内容**（前 50 行即可），了解笔记主题
2. **展示摘要**给用户，包括文件名和内容概要
3. **用 AskUserQuestion 询问**目标分区：

```
问题：这篇笔记「{文件名}」应该归档到哪里？
选项：
- 10-Projects（有截止日期的项目）
- 20-Areas（持续负责的领域）
- 30-Resources（学习和参考资料）
- 40-Archive（归档）
```

4. 用户选择分区后，**列出该分区下已有的子目录**，让用户选择放入哪个子目录（或创建新的）
5. **执行移动**：`mv 00-Inbox/{文件} {目标路径}/`

### 3. 更新导航

所有文件处理完毕后：

- 如果目标路径有对应的 MOC 文件，读取 `rules/moc-sync.md` 执行 MOC 同步
- 如果有新项目或新分区，读取 `rules/home-sync.md` 更新首页

### 4. 汇报结果

给出处理摘要：
```
已处理 N 篇笔记：
- 「xxx.md」→ 10-Projects/xxx/
- 「yyy.md」→ 30-Resources/yyy/
```
